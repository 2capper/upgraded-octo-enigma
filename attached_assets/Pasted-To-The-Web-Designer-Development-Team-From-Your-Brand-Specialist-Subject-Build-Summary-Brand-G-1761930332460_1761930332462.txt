2. Multi-Tenant UI/UX Strategy (The "Tenant Space")
This is the most critical component of the build. We must allow tenant organizations (like the OBA) to customize their own tournament spaces without breaking the core app experience.

The "App Shell" vs. "Tenant Space" Model:

The "App Shell" (NON-Customizable):

The top app header (with the GameDay Guru "G" logo).

The bottom navigation bar.

App-level menus (Settings, Profile, etc.).

Core app-level buttons (e.g., "Create New Tournament").

All core typography (font, size, weight).

The "Tenant Space" (Customizable by Org Admins):

This refers to elements inside a specific tournament's card or pages.

We will provide admins a simple panel to input three variables:

Tenant Logo (Image Upload)

Tenant Primary Color (Color Picker)

Tenant Secondary Color (Color Picker)

How to Apply Tenant Variables:

On the "My Tournaments" Dashboard:

Each tournament card is a "themed" component.

It must display the Tenant Logo.

It must use the Tenant Primary Color as an accent (e.g., the card's top border or header bar).

On Tournament-Specific Pages (e.g., "OBA Championship Schedule"):

A banner area at the top can display the Tenant Logo and/or Tenant Primary Color.

Tenant-specific buttons (e.g., "Log OBA Score," "View OBA Rules") should use the Tenant Primary or Secondary Color.

Links, progress bars, and other indicators within this space should also adopt the tenant's colors.
